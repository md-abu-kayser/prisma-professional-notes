# Contributing

Thanks for considering a contribution to Prisma Professional Notes!

## Ways to contribute

- Fill in a stubbed note with real explanations and examples
- Fix inaccuracies or outdated Prisma API references
- Add a new real-world project walkthrough
- Improve a code snippet in `17-code-snippets/`

## Guidelines

1. Keep one topic per file, matching the existing folder structure.
2. Prefer small, focused pull requests over large rewrites.
3. Include a minimal runnable example where relevant (schema snippet, query,
   or code block).
4. Link out to the official [Prisma docs](https://www.prisma.io/docs) for
   anything version-specific, since APIs evolve.

## Workflow

1. Fork the repo and create a branch: `git checkout -b notes/topic-name`
2. Make your changes.
3. Open a PR using the provided template.
